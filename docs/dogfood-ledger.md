# Dogfood Ledger

This ledger tracks the public-alpha live-use gate. Fake-tool hardening proves
the safety invariants; dogfood proves the product remains useful against real
Xcode projects, real simulator runtimes, and real failure modes.

## Failure Injection Plan

Use disposable project/profile state roots for these cases:

| Scenario | Expected Result |
| --- | --- |
| Wrong simulator UDID | `runner_bootstrap_failure`, no simulator mutation, logs and run metadata preserved. |
| Missing test plan | Doctor fails before submit with a non-auto-fixable project check. |
| Xcode/runtime mismatch | Doctor fails `global.iphonesimulator_runtime_compatible`. |
| Build failure | Job fails as `build_failure`, no test phase starts. |
| Test failure | Job fails as `test_failure`, `.xcresult` and logs remain discoverable. |
| Test timeout | Job fails as `test_timeout`, simulator lease released. |
| Cancel during boot | Job reports `canceled`, owned boot process terminated. |
| Cancel during build | Job reports `canceled`, owned `xcodebuild` terminated. |
| Cancel during test | Job reports `canceled`, owned test `xcodebuild` terminated. |
| Kill worker during build/test | Restart reconciles stale running job and queue continues. |

## Update Rules

- Add one row per real run or failure injection.
- Record the state root or report path when it is useful evidence.
- Count an incident as S0 if it risks unrelated developer state, false success,
  artifact loss, duplicate execution, permanent queue deadlock, or broad
  CoreSimulator cleanup without explicit confirmation.
- Do not count fake-tool matrix rows as real local jobs.
